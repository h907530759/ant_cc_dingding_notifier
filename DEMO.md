# 功能演示：Hook 事件选择

## 新增功能

现在配置工具会让您**交互式选择**要启用哪些 Hook 事件！

## 使用示例

```bash
./run.sh setup
```

## 配置流程

### 1. 配置钉钉机器人

输入 Webhook URL 和加签密钥（可选）

### 2. 选择要启用的事件 🎯

系统会显示所有可用的事件：

```
┏━━━━┳━━━━━━━━━━━━━┳━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┳━━━━━━┓
┃ 编号 ┃ 事件         ┃ 说明                           ┃ 推荐 ┃
┡━━━━╇━━━━━━━━━━━━━╇━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━╇━━━━━━┩
│ 1   │ pre_tool_use │ 检测敏感操作（sudo、rm、docker等）│ ⭐ 推荐│
│ 2   │ post_tool_use│ 记录执行结果和错误              │ ⭐ 推荐│
│ 3   │ stop         │ 任务完成通知                    │ ⭐ 推荐│
│ 4   │ notification │ 处理权限请求                    │ 可选  │

```

### 3. 输入您的选择

```
请选择要启用的事件（输入编号，用逗号分隔，如: 1,2,3）
提示: 直接回车表示启用所有推荐事件（1,2,3），输入 0 表示全部禁用

您的选择 (1,2,3): 
```

### 4. 选择方式

- **直接回车**: 启用推荐事件（1,2,3）
- **输入 `1,2,3`**: 启用推荐事件
- **输入 `1,2,3,4`**: 启用所有事件
- **输入 `1,3`**: 只启用 pre_tool_use 和 stop
- **输入 `0`**: 禁用所有事件
- **输入 `2,4`**: 只启用 post_tool_use 和 notification

### 5. 确认配置

系统会显示您的最终选择：

```
最终配置:
  pre_tool_use: ✓ 启用
  post_tool_use: ✓ 启用
  stop: ✓ 启用
  notification: ✗ 禁用
```

## 事件说明

### 1. pre_tool_use（推荐）

**触发时机**: 工具使用前

**用途**: 检测敏感操作

**通知内容**: 
- 检测到敏感操作（sudo、rm -、git push、docker、kubectl、npm publish）
- 显示工具名称和输入内容
- 提醒用户在终端确认

**推荐理由**: ⭐⭐⭐ 安全重要，防止误操作

### 2. post_tool_use（推荐）

**触发时机**: 工具使用后

**用途**: 记录执行结果和错误

**通知内容**:
- 仅在发生错误时通知
- 显示错误的项目和工具

**推荐理由**: ⭐⭐⭐ 及时发现错误，不会产生过多通知

### 3. stop（推荐）

**触发时机**: Claude Code 停止工作

**用途**: 任务完成通知

**通知内容**:
- 项目名称
- 完成时间
- 状态摘要

**推荐理由**: ⭐⭐⭐ 了解任务进度，重要的里程碑通知

### 4. notification（可选）

**触发时机**: 权限请求和提示

**用途**: 处理权限请求

**通知内容**:
- 权限请求通知
- 提示在终端确认

**推荐理由**: ⭐⭐ 可能产生较多通知，按需启用

## 推荐配置

### 日常开发（推荐）

```
选择: 1,2,3
```

启用：pre_tool_use, post_tool_use, stop

**优点**: 
- ✅ 安全防护
- ✅ 错误提醒
- ✅ 任务完成通知
- ✅ 不会过多打扰

### 调试模式

```
选择: 1,2,3,4
```

启用所有事件

**优点**:
- ✅ 全面监控
- ✅ 所有操作都有反馈

**缺点**:
- ❌ 通知较多

### 最小化配置

```
选择: 1,3
```

只启用：pre_tool_use, stop

**优点**:
- ✅ 安全防护
- ✅ 任务完成通知
- ✅ 最少打扰

### 仅安全监控

```
选择: 1
```

只启用：pre_tool_use

**优点**:
- ✅ 纯安全防护
- ✅ 最少通知

## 修改配置

随时可以重新运行 setup 来修改选择：

```bash
./run.sh setup
```

或者直接编辑配置文件：

```bash
# 编辑配置
vim ~/.claude-dingtalk/config.yaml

# 找到 events 部分，修改 enabled 值
events:
  pre_tool_use:
    enabled: true   # true = 启用, false = 禁用
  post_tool_use:
    enabled: false
  stop:
    enabled: true
  notification:
    enabled: false
```

## 测试通知

配置完成后，测试每个事件是否正常工作：

```bash
# 测试基本通知功能
./run.sh test

# 查看当前配置
./run.sh status
```
